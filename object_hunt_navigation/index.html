About this page

This page offers information about all used classes, namespaces and header files for the navigation process. The navigation process tries to establish a TCP connection to the base process and starts requesting the ultra sonic sensors immediately. Depending on the response it computes a way to go and sends a motor command to the base process. Consequemtly it enters a loop of requesting sensor measuring, processing and sharing measurement responds and sending motor commands.

While the navigation is running, a separated thread listens for connection on a defined port (19002). It handles how new connection are handled, advises the navigation thread in case of an event and removes closed connections. All connection related methods are implemented in the ConnectionManager class.
